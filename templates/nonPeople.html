Non-People Accounts

Overview

A non-people account, or a service account, is a non-human privileged account that an operating system uses to run applications, automated services, virtual machine instances, and other processes. These accounts can be privileged local or domain accounts; in some cases, they may have domain administrative privileges. They provide a way to assign an identity and permissions to a computer program or process that performs a specialized task.

This website aims to streamline the process of accessing information on non-people or service accounts, providing a convenient and efficient platform for users to enhance their understanding and make informed decisions in managing these accounts effectively.

How do I request a Non-People Account?

Before requesting a non-people account, assess your information systems integration requirements, including the rights they should have.

  • For Active Directory submit One Domain Application or Service Account (Active Directory)
  • For Enterprise Directory submit NonPeople LDAP Account
  • Contact the application owner to create an account in any other system

Tier 0 and 1 need to be registered into CyberArk.

What is a Non-People Account Tier?

Account Tier classification defines the risk of the account based on its privileges and scope according to IT-SOP-400/ IT-WI-4014 Controls for Access Credentials.

A non-people account must have one of the below classifications assigned:

  • Tier 0 - Enterprise Privileged: Accounts with root or admin access to all or many systems across the enterprise
  • Tier 1 - Privileged: Accounts with root or admin access to a limited set of applications, servers, databases, or other IT infrastructure that support a SOX application or other application with a specific regulatory requirement
  • Tier 2 - Limited Privileged: Non-SOX application accounts with elevated access to an application, such as workstation administrative access to a single workstation
  • Tier 3 - No Elevated Privilege: Accounts do not have access to privileged systems and data

How do I assign a Tier to my Non-People account?

  • For Active Directory and Enterprise Directory go to Search for my non-people accounts in LDAP
    1. Click in the Tier field for the account you want to update
    2. In the dialog, set the Tier to a value of 0, 1, 2 or 3
    3. Click Save
  • For CyberArk submit General Request to APP SUPPORT ACN PRIVILEGED ACCESS MANAGEMENT group
  • Contact the application owner to assign an account Tier in any other system

How do I change the owner of a Non-People Account?

  • For Active Directory and Enterprise Directory go to Search for my non-people accounts in LDAP
  • For CyberArk go to Search for non-people accounts in CyberArk
    1. Find Non-People Account
    2. Click on Safe Name to see what groups own the safe
    3. Submit Access to Active Directory Group request to be added or deleted from the group
  • Contact the application owner to assign an owner in any other system.

How do I change the password of a Non-People Account?

  • For Enterprise Directory go to  LDAP Nonpeople Accounts
  • For Active Directory submit General Request to ISSA ACTIVE DIRECTORY ENG group
  • For CyberArk fully onboarded accounts, password rotation rules are set by the CyberArk admin in the platform policies for each account.
    For example, passwords may be rotated every 90 days. Please go to Access PAM (CyberArk) Training Guide for more information.
  • Contact the application owner to change the password in any other system.

How do I register a Non-People Account in CyberArk Privileged Access Management?

Account owners must register a non-people account classified as Tier 0 or Tier 1 in the CyberArk Privileged Access Management (PAM) system.

Follow these steps to register a non-people account in the CyberArk.

  • Submit CyberArk Privileged Access Management (PAM) Safe Creationrequest
  • Submit CyberArk Privileged Access Management (PAM) Account Onboarding request

For support, contact the CyberArk PAM team at MG-IMSS-PAM-PROD-APP-ADMINS@bms.com
